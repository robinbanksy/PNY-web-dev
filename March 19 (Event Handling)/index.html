Event Handling

Events

When users interact with our web pages, they generate events. Examples include clicking a button, submitting a form, or hovering over an element. We can handle these events using JavaScript to add functionality to our web pages.

Event Listeners

In the previous class, we learned about in-HTML event handlers. However, it's generally better practice to use JavaScript event listeners for more flexibility and control.

For example, to handle a button click, we can use the following code:

const button = document.querySelector('button');
button.addEventListener('click', function() {
     doSomething(); // Regular JavaScript code goes here
});

In the example, we first get a reference to the button element, and then use the addEventListener method to attach an event listener to it.

The addEventListener method takes two arguments: the event type (e.g., 'click') and a callback function. The callback function is executed when the event is triggered.

We usually use the anonymous function syntax for the callback function, as it allows us to define the function inline without giving it a name.

Event Propagation

When an event occurs on an element, it doesn't just affect that element. It can also affect its parent elements. This is called event propagation.

The process by which events first are handled on the target element, and then propagate to parent elements, is called bubbling. Observe the following example, where both the parent and child elements have event listeners:


const parent = document.querySelector('.parent');
const child = document.querySelector('.child');
parent.addEventListener('click', function() {
     alert('Parent clicked');
});
child.addEventListener('click', function() {
     alert('Child clicked');
});

Child

Notice that when you click on the child element, both alerts are triggered due to event propagation. When the parent element is clicked, only its own alert is triggered.

In addition to bubbling, there is also a capturing phase, where events are handled from the top down (from the parent to the child). By default, event listeners are set to handle events during the bubbling phase, but you can specify that an event listener should handle events during the capturing phase by passing a third argument as true:

element.addEventListener('click', function() {
     // Handle click event
}, true); // true for capturing phase

The capturing phase is less commonly used, but it can be useful in specific scenarios where you want to handle events before they reach the target element.
